## 低代码AI平台是什么？

简单说：**不写代码（或少写代码），通过可视化界面搭建AI应用**。

传统方式：
```python
# 写一堆代码...
from langchain import ...
from openai import ...
# 配置模型
# 写提示词
# 处理输入输出
# 搭建API
# 部署上线
# ... 几百行代码
```

低代码平台：
```
拖入"输入节点" → 拖入"大模型节点" → 拖入"输出节点" → 连线 → 发布
```

**5分钟搞定一个AI客服、知识库问答、智能助手**。

## 三大平台概览

| 平台 | 背景 | 特点 | 定位 |
|:---|:---|:---|:---|

## Dify：全能型选手

### 简介

Dify 是目前功能最全面的开源低代码AI平台，由LangGenius团队开发。

- 官网：https://dify.ai
- GitHub：https://github.com/langgenius/dify
- Stars：50k+

### 核心功能

**1. 可视化工作流**

```
┌─────────┐    ┌─────────┐    ┌─────────┐
│  开始   │ →  │ LLM调用 │ →  │ 条件判断│
└─────────┘    └─────────┘    └─────────┘
                                  ↓
                    ┌─────────────┴─────────────┐
                    ↓                           ↓
              ┌─────────┐                 ┌─────────┐
              │ 知识库  │                 │ 代码执行│
              └─────────┘                 └─────────┘
```

支持节点：
- LLM（GPT/Claude/Qwen等）
- 知识库检索
- 条件分支
- 代码执行（Python/JavaScript）
- HTTP请求
- 变量赋值
- 迭代循环

**2. 知识库（RAG）**

```
上传文档 → 自动分块 → 向量化 → 存储 → 检索增强
```

支持格式：PDF、Word、Markdown、TXT、网页链接

**3. 多模型支持**

```yaml
支持的模型：
- OpenAI: GPT-4o, GPT-4, GPT-3.5
- Anthropic: Claude 3.5, Claude 3
- Google: Gemini Pro
- 国产: 通义千问, 文心一言, DeepSeek, Moonshot
- 本地: Ollama, LocalAI
```

**4. API & 嵌入**

```python
# 调用你搭建的应用
import requests

response = requests.post(
    'https://api.dify.ai/v1/chat-messages',
    headers={'Authorization': 'Bearer your-api-key'},
    json={
        'inputs': {},
        'query': '你好',
        'user': 'user-123'
    }
)
```

还可以一键嵌入到网站：
```html
<script src="https://dify.ai/embed.js"></script>
```

### 实战：搭建客服机器人

**Step 1：创建应用**

选择"聊天助手" → 输入应用名称

**Step 2：配置提示词**

```
你是一名专业的电商客服，名叫小智。

职责：
1. 回答商品相关问题
2. 处理订单查询
3. 解决售后问题

规则：
- 语气友好亲切
- 不知道的问题说"我帮您转接人工客服"
- 涉及退款说"请提供订单号，我帮您查询"
```

**Step 3：添加知识库**

上传：
- 产品手册.pdf
- FAQ文档.md
- 退换货政策.docx

**Step 4：测试发布**

在预览窗口测试对话 → 满意后点击"发布"

**Step 5：获取API**

复制API Key → 集成到你的网站/App

### Dify 优缺点

**优点**：
- 开源可私有化部署
- 功能最全面
- 工作流设计灵活
- 社区活跃，更新快
- 支持团队协作

**缺点**：
- 学习曲线略陡
- 部分高级功能需要编码
- 私有化部署需要运维能力

## Coze：简单好用派

### 简介

Coze（扣子）是字节跳动推出的AI Bot开发平台。

- 国际版：https://coze.com
- 国内版：https://coze.cn

### 核心功能

**1. Bot创建（超简单）**

```
输入Bot名字 → 写一段人设 → 选择模型 → 完成！
```

**2. 插件市场**

Coze最大的亮点是丰富的插件生态：

```
内置插件：
- 联网搜索
- 图片生成（DALL-E/MidJourney）
- 代码执行
- 文档解析
- 数据分析
...200+插件

第三方插件：
- 天气查询
- 股票行情
- 快递追踪
- 翻译服务
- ...
```

**3. 工作流（Flow）**

可视化编排，但比Dify简单：

```
触发器 → 处理节点 → 输出
```

**4. 多平台发布**

一键发布到：
- 网页
- Discord
- Telegram
- Slack
- 飞书
- 微信（公众号/小程序）

### 实战：搭建新闻摘要Bot

**Step 1：创建Bot**

名称：每日新闻助手

人设提示词：
```
你是一个新闻摘要助手。
用户发送新闻链接，你负责：
1. 提取关键信息
2. 生成3句话摘要
3. 给出重要性评分（1-5星）
```

**Step 2：添加插件**

- 启用"网页读取"插件
- 启用"联网搜索"插件

**Step 3：配置工作流**

```
用户输入URL 
    ↓
[网页读取插件] 获取内容
    ↓
[LLM节点] 生成摘要
    ↓
返回结果
```

**Step 4：测试发布**

发布到 Discord/Telegram，随时随地用！

### Coze 优缺点

**优点**：
- 上手极简，5分钟搞定
- 插件生态丰富
- 多平台发布方便
- 免费额度慷慨
- 支持豆包模型（免费）

**缺点**：
- 不开源，无法私有化
- 复杂场景支持有限
- 数据存储在云端
- 企业级功能较弱

## FastGPT：知识库专家

### 简介

FastGPT 专注于知识库问答场景，是构建企业知识库的首选。

- 官网：https://fastgpt.in
- GitHub：https://github.com/labring/FastGPT
- Stars：20k+

### 核心功能

**1. 高级RAG**

FastGPT的知识库检索做得最细致：

```
文档上传
    ↓
智能分块（支持多种策略）
    ↓
多路召回（向量+关键词+全文）
    ↓
重排序（Rerank）
    ↓
上下文压缩
    ↓
生成回答
```

**2. 对话日志分析**

```
┌─────────────────────────────────────────┐
│  问题统计    回答质量    用户满意度      │
│  ████████    ████████    ████████       │
│                                          │
│  高频问题 Top 10                         │
│  1. 如何退款？ (234次)                   │
│  2. 发货时间？ (189次)                   │
│  ...                                     │
└─────────────────────────────────────────┘
```

**3. 应用编排**

支持复杂的对话流程设计：

- 意图识别
- 多轮对话
- 表单收集
- 人工转接

### 实战：搭建企业知识库

**Step 1：上传文档**

支持批量上传：
- 员工手册
- 产品文档
- 技术规范
- 历史工单

**Step 2：配置索引**

```yaml
分块策略: 按段落
块大小: 500字
重叠: 50字
向量模型: text-embedding-3-small
```

**Step 3：配置检索**

```yaml
召回方式: 混合检索
召回数量: 5
相似度阈值: 0.7
启用重排序: 是
```

**Step 4：设置问答**

```
系统提示词：
你是公司的AI助手。基于知识库内容回答问题。
如果知识库没有相关信息，请说"这个问题我需要确认后回复您"。
```

**Step 5：嵌入网站**

```html
<iframe src="https://fastgpt.in/chat/xxx" width="400" height="600"></iframe>
```

### FastGPT 优缺点

**优点**：
- 知识库功能最强
- RAG效果最好
- 开源可私有化
- 对话分析完善
- 部署简单（Docker一键）

**缺点**：
- 工作流相对简单
- 插件生态不如Coze
- 非知识库场景较弱

## 快速上手指南

### Dify 5分钟上手

```bash
# Docker 部署
git clone https://github.com/langgenius/dify.git
cd dify/docker
docker compose up -d

# 访问 http://localhost:3000
```

### Coze 5分钟上手

1. 访问 https://coze.cn
2. 登录（抖音/飞书账号）
3. 点击"创建Bot"
4. 输入名称和提示词
5. 发布！

### FastGPT 5分钟上手

```bash
# Docker 一键部署
curl -o docker-compose.yml https://raw.githubusercontent.com/labring/FastGPT/main/files/deploy/fastgpt/docker-compose.yml
docker compose up -d

# 访问 http://localhost:3000
```
